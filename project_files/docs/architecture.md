# Архитектура MVP

## Задача

Пользователь передает XLSX-выгрузку из SAP. Сервис формирует новый XLSX, в котором к исходным колонкам `A:S` добавлены расчетные колонки `T:AA` с Excel-формулами.

Файлы не должны храниться в будущей продуктовой реализации: обработка должна быть одноразовой в рамках запроса.

## MVP-решение

Основной текущий MVP - browser-only приложение:

```text
input.xlsx
  -> browser File API
  -> ExcelJS transform in browser
  -> generated output.xlsx download
```

Python/Flask версия оставлена как fallback и эталон для проверок:

```text
input.xlsx
  -> sap_xlsx_transformer
  -> output.xlsx
```

Компоненты:

- `browser_app/src/transformer.ts` - основная browser-only логика;
- `browser_app/src/main.ts` - drag-and-drop UI и пакетная обработка;
- `browser_app/test-browser-transform.mjs` - проверка browser-only трансформера на реальных XLSX;
- `app/sap_xlsx_transformer/` - Python fallback.

## Почему правила отдельно

Формулы и условия будут уточняться. Поэтому код разделен так:

- техническая работа с Excel живет в `browser_app/src/transformer.ts`;
- спорные условия описаны в `docs/dependencies.md`.

Если меняется ставка КК, дата расчета, набор расчетных колонок или поведение фильтров, правится небольшой участок в `browser_app/src/transformer.ts` и тест `test-browser-transform.mjs`.

## Обработка данных

Browser-only MVP:

- читает XLSX прямо в браузере;
- удаляет старую расчетную область правее `S`;
- добавляет расчетные колонки `T:W`;
- добавляет условные формулы `X:AA` только для строк `DR`, где `P` заполнена и `T > 0`;
- применяет формулы только к строкам `DR`;
- создает скачиваемый XLSX без отправки файла на сервер.

Статическое развертывание:

```text
subdomain
  -> nginx / CDN
  -> browser_app/dist/
```

Допустимая статистика без хранения содержимого:

- дата обработки;
- пользователь или подразделение;
- имя файла или хэш;
- число строк;
- число строк `DR`;
- статус обработки;
- версия правил.
