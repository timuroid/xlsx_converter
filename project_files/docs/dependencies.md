# Выявленные зависимости в примере `100019128.XLSX`

## Входная часть

SAP-выгрузка занимает колонки `A:S`.

Ключевые входные колонки:

- `O` - `Дата платежа`;
- `P` - `Дата выравнивания`;
- `Q` - `Вид документа`;
- `R` - `Сумма в ВВ`.

Расчет применяется только к строкам, где `Q = DR`.

## Расчетные колонки актуального MVP

Актуальный browser-only MVP добавляет только подтвержденные по письму колонки `T:W`.

```text
T  Дней просрочки по КК (календарные)
U  % КК
V  Сумма КК
W  Сумма закрытия
```

В `AB1` хранится дата расчета. Она используется только для расчета `T`, если `P` пустая.

## Базовые правила

Для строк `DR`:

```text
если P заполнена:
  T = MAX(0, P - O)
иначе:
  T = MAX(0, $AB$1 - O)

U = 0.001
V = U * T * R
W = V + R
```

## X:AA: только заголовки

В выходной файл добавляются заголовки колонок `X:AA`, но значения и формулы в них не проставляются:

```text
X  К доплате после пересчета платежа:
Y  к доплате в том числе по основному долгу
Z  к доплате в том числе по КК
AA Количество дней просрочки долга после пересчета
```

Их правила не описаны в письме заказчика и были выведены только эмпирически из примера. До отдельного подтверждения заказчиком эти колонки остаются пустыми.

Ранее предполагаемая логика, требующая подтверждения:

```text
условие: Q = DR, P заполнена, T > 0
AA = MAX(0, $AB$1 - P)
Y  = V
Z  = AA * Y * U
X  = Y + Z
```

## Открытые вопросы

- Нужны ли когда-либо формулы в `X:AA`; если да, нужны точные правила от заказчика.
- Нужно ли сохранять или сбрасывать фильтры из исходной SAP-выгрузки.
- Нужно ли добавлять строку итогов `SUBTOTAL`.
- Нужно ли оставлять `AB1` видимой или скрывать служебную колонку.
