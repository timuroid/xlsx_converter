# Правила проекта

## Контекст

Проект относится к IntBiS / Уральская сталь. Цель проекта - простой XLSX-трансформер для SAP-выгрузок: пользователь передает исходный XLSX, получает новый XLSX с расчетными колонками и формулами.

## Рабочие правила

- Рабочий язык проекта: русский.
- Не хранить пользовательские входные и выходные XLSX в будущей продуктовой реализации.
- В репозитории можно хранить только тестовые примеры, явно добавленные как проектные источники.
- Формулы и условия расчета держать отдельно от основного кода.
- Новые правила добавлять так, чтобы их можно было изменить точечно после голосового/текстового уточнения пользователя.
- Предпочитать простую реализацию без лишней инфраструктуры.
- Основной продуктовый путь - browser-only приложение в `project_files/browser_app/`.
- Python MVP на `openpyxl` в `project_files/app/` считать fallback/эталоном для проверки расчетной логики.
- Проверять изменения на приложенном файле и на минимальных edge-case сценариях.
- Перед публикацией в GitLab не добавлять реальные рабочие XLSX без отдельного подтверждения пользователя.

## Формат отчетов

Отчеты пользователю писать кратко:

- что изменено;
- какие правила или формулы затронуты;
- чем проверено;
- какие вопросы остались открытыми.

## Ограничения

- Не менять материалы в `/Работа/IntBiS/02_Общая докементация/spec/` без явного разрешения пользователя.
- Не удалять исходники, документы, модели и результаты работы без явной просьбы пользователя.
- Не складывать рабочие файлы в корень `/Users/timuroid/Desktop/work`.
